Feature spec · 2026-05-27
Warn users about rewards expiring within 7 days.
Currently rewards silently expire — users open the app, find a perk gone, and email support. This change surfaces upcoming expiry in the rewards list and (for users with push enabled) sends a 3-day-out notification. Expired rewards move into a collapsed "Expired" section instead of being hidden.

01 Acceptance criteria
4 criteria ▸
CRIT-1
Given a reward expiring within 7 days, when the user opens the rewards list, then a warning banner appears anchored to that reward.
CRIT-2
Given a user with push notifications enabled and a reward expiring in exactly 3 days, when the daily expiry cron runs, then a push notification is queued via the existing notification service.
CRIT-3
Given a reward past its expiry date, when the rewards list loads, then the reward appears under a collapsed "Expired" section rather than the main list.
CRIT-4
Given the reward-expiry-warnings flag is off, when any of the above runs, then behavior matches current production exactly.
02 Non-goals
3 items ▸
NOT Email expiry warnings. Push only for this iteration.
NOT User-configurable warning thresholds (e.g. "warn me 14 days out").
NOT Reactivating expired rewards. Expired stays expired.

CRIT-1
Banner appears
Reward expiring in 6 days → banner renders with "Expires in 6 days".
Reward already expired → banner does NOT render (it's in the Expired section).
Mock the date to make every reward "expiring soon" — banner should NOT render on rewards 8+ days out.
CRIT-2
Push notif
User with push enabled, reward expiring exactly 3 days out → one push queued.
User with push disabled → no push queued, no error.
Cron ran twice in one day — second run should not double-send.
CRIT-3
Expired section
Past-expiry reward → renders under "Expired", section collapsed by default.
No expired rewards → "Expired" section does not render at all.
Reward expiring today at 23:59 — should still be active, not in Expired.
CRIT-4
Flag off
Flag off → list renders identical to current production snapshot.
Flag flips off mid-session → next render reverts cleanly.
Flag check evaluates to "expired" (falsy) → must not pass as "on".

06 Open questions
3 unresolved · before /build ▸
Unresolved before build
1. Timezone. Do we evaluate "within 7 days" in the user's local timezone or UTC? Affects users near the boundary at midnight.
2. Batching. If a user has 5 rewards all expiring within 3 days, do they get 5 pushes or one batched push?
3. Banner component. Does the design system already have a warning banner pattern we should reuse, or do we ship a new one?
